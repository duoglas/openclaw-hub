## 今日主题：安全不是“出事再修”，而是默认最小权限

### 1) 先收权限，再谈效率
- 不常用渠道先关闭
- 不需要的插件先禁用
- 能本地完成的，不走外发

### 2) 定期做审计
```bash
openclaw security audit
openclaw security audit --deep
```
重点关注：
- 可疑执行（shell/child_process）
- 文件读取 + 网络发送组合
- 反向代理/来源信任配置

### 3) 外部动作保持“人工闸门”
发消息、发邮件、公开发布前，维持人工确认是最稳妥做法。

## 今日结论
- **能长期跑的自动化，前提是可控。**
- 对个人用户，最有效的安全策略是“少开、慢开、可回滚”。
- 明天跟踪点：新增插件风险、暴露面变化、审计告警趋势。
