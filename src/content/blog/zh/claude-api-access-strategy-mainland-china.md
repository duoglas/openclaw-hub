对于中国大陆团队，Claude API 接入不是“单点技巧”，而是“稳定架构 + 合规边界 + fallback 机制”的组合。

## 核心策略

1. **稳定的出海基础设施**
   - 网关部署在稳定的海外区域
   - API Key 仅保留在服务端

2. **做一层模型抽象**
   - 统一走你自己的 API 层
   - 业务逻辑不要硬绑定单一模型厂商

3. **设计 fallback 策略**
   - 主模型：Claude（高质量任务）
   - 备份模型：其他一线模型（保障连续性）
   - 针对不同请求设超时与重试预算

4. **按敏感级别分流数据**
   - P0：禁止外部模型调用
   - P1：仅可脱敏数据
   - P2：经审批后可完整上下文

## 常见误区

只追求“先能连上”，忽视监控、成本和故障切换，后期会导致运营不稳定。
