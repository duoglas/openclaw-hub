## 今日 OpenClaw 要点

### 1) 保持频道连接稳定
- 用 `openclaw status` 检查各频道状态

```
频道
┌──────────┬─────────┬────────┬──────────────────────────────┐
│ 频道     │ 启用    │ 状态   │ 详情                          │
├──────────┼─────────┼────────┼──────────────────────────────┤
│ Telegram │ ON      │ OK     │ token 已配置                  │
│ Slack    │ ON      │ OK     │ token 正常                    │
│ QQ Bot   │ ON      │ OK     │ 已配置                        │
│ 企业微信 │ ON      │ OK     │ 已配置                        │
└──────────┴─────────┴────────┴──────────────────────────────┘
```

确保所有频道状态都是 OK。如果某个频道断连，检查 Token 是否过期或网络是否通畅。

### 2) 合理设置模型回退链
- 避免相邻回退使用同一提供商 —— 限流时会一起挂。
- 推荐交替排列不同提供商，例如 Anthropic → OpenAI → MiniMax → Google。

### 3) 保持上下文健康
- 尽量使用短轮次提示，避免单个对话积累过多 token。
- 定期执行上下文压缩（compaction），释放窗口空间。
- 长任务考虑拆分成子代理（subagent）。

### 4) 安全快速检查
- 定期运行 `openclaw security audit`

```
安全审计摘要：2 严重 · 2 警告 · 1 信息
```

**严重项关注**：
- 检查第三方插件是否存在危险代码模式（如 `child_process` 调用）
- 发现可疑的环境变量访问 + 网络发送行为时，审查插件源码

**警告项**：
- 如果通过反向代理暴露控制面板，务必配置 `trustedProxies`
- 检查文件读取 + 网络发送的组合行为，确认是否为正常功能

建议每周至少做一次安全审计，用 `--deep` 参数进行深度扫描。
